# Contributing to SafetyDesk AI

Thank you for your interest in contributing! This document provides guidelines for contributing to the SafetyDesk AI project.

## Code of Conduct

Be respectful, inclusive, and professional. We maintain a harassment-free community.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a feature branch: `git checkout -b feature/your-feature`
4. Make your changes
5. Write or update tests
6. Commit with clear messages
7. Push to your fork and submit a Pull Request

## Development Guidelines

### Branching Strategy
- `main` — Production-ready code
- `develop` — Integration branch for features
- `feature/` — Feature branches
- `bugfix/` — Bug fix branches
- `hotfix/` — Production hotfixes

### Commit Messages
Format: `[TYPE] Description`
- `[FEAT]` New feature
- `[FIX]` Bug fix
- `[DOCS]` Documentation
- `[REFACTOR]` Code refactoring
- `[TEST]` Tests
- `[CHORE]` Build, dependencies

Example: `[FEAT] Add OSHA 300 log export functionality`

### Code Style
- Backend: ESLint + Prettier
- Frontend: ESLint + Prettier + Stylelint

### Testing Requirements
- Unit tests for new functions
- Integration tests for API endpoints
- Minimum 80% code coverage

## Pull Request Process

1. Update documentation if needed
2. Add/update tests
3. Ensure CI/CD passes
4. Request review from maintainers
5. Address feedback
6. Squash commits before merge

## Reporting Issues

- Use GitHub Issues
- Include clear description, steps to reproduce, expected vs actual behavior

## Questions?

Reach out to the maintainers or post in discussions.
